Аутентификация и Авторизация

Прежде чем изменить метод, нужно сохранить его интерфейс. Метод должен возвращать те же значения, которые возвращал до этого. Public static function findByUsername ($username) {if($user = Users::findOne(['login' => $username])){return new static (['id' => $user->id, 'username' => $user->login, 'password' => $user->password]);} return null}; /// new Static() - экземпляр текущего объекта. Выше можно было использовать $users->toArray При инициализации объекта, но у нас разные названия в БД и объекте. Поэтому мы назначили всё вручную Однако мы можем переопределить поля объекта, получаемого из БД, переопределив в модели метод fields: public function fields() {return ['id', => 'username'=>'login', 'password']}.///// И теперь мы можем писать так new Static($user->toArray()); //// Единственный недостаток того, что мы сделали, login всегда будет заменяться на username, а это не всегда может быть нужно. Чтобы такого не было, мы можем применить сценарии. Это ещё одна из функциональностей моделей. У любой модели есть свойство scenario - это текущий сценарий, который по умолчанию равен dafault Установка сценария. $user-> setScenario('auth'). Это флаг, показывающий, текущий путь развития логики. Хорошая практика - записывать флаги в константы. Const SCENARIO_AUTH = 'auth'. И тогда запишем $user->setScenario(Users::SCENARIO_AUTH). В первую очередь сценарии используются для управления правилами валидации. Т.е. мы можем переопределить метод scenarios и в нём мы можем указать при каком сценарии какие поля проверять public function scenarios(){return ['auth' => ['login', 'pass']]}. Т.е. при сценарии auth проверяй только login и pass. Сработает при вызове метода validate. Также можно задать не только валидацию полей, но и конкретных правил. Вот например правило будет применяться только при нашем сценарии public function rules() {return [[['login', 'password'], 'required', 'on' => static::SCENARIO_AUTH]]}; Также можем поставить этот флаг в метод fields. public function fields() {if ($this->scenario == static::SCENARIO_AUTH) {return ['id', => 'username'=>'login', 'password']}return parent::fields()} Работа с паролем: \YII::$app->security->generatePasswordHash().
